# Contributing

Thanks for your interest in contributing.

## Development Setup

1. Fork the repository.
2. Create a branch from `master`.
3. Run the app locally using either Docker or non-Docker setup from the README.
4. Make focused changes with clear commit messages.

## Pull Request Guidelines

- Keep PRs small and scoped.
- Update documentation when behavior changes.
- Include reproduction steps for bug fixes.
- Ensure the app builds successfully for both backend and frontend.

## Coding Expectations

- Prefer TypeScript types over `any`.
- Keep API changes backward-compatible when possible.
- Add tests when test infrastructure is introduced.

## Reporting Issues

Use GitHub Issues with:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment details (OS, browser, Node version)
